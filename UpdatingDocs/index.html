# How to update the VuePress documentation

# Adding documentation for a new component

  • Create a new directory for the component in sbc-common-components/docs
  • In the new directory, create a README.md describing the component and how to use it
sbc-common-components
├─ docs
│  ├─ .vuepress
|  ├─ Header
|  ├─ Footer
│  └─ NewComponent
│     └─ README.md
  • To ensure a link to the documentation is visible in the sidebar, open .vuepress/config.js and add the path to your new documentation in the array children for the sidebar group Components.
// in .vuepress/config.js
module.exports = {
  base: '/sbc-common-components/',
  title: 'SBC Common components',

  themeConfig: {
    sidebar: [
      {
        title: 'Components',
        collapsable: false,
        children: [
          '/Header/',
          '/Footer/',
          '/NewComponent/'
        ]
      }
    ]
  }
}

# Publishing changes to the documentation

  • Ensure you have write permissions to bcgov/sbc-common-components
  • Run the script deployvuepress.sh to publish your changes to the documentation
